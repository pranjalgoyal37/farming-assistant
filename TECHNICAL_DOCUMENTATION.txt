PROJECT OVERVIEW
================================================================================
FarmAssist is an AI-powered farming assistant web application that helps 
farmers make data-driven decisions for crop selection, fertilizer guidance, 
disease detection, and weather monitoring. The core feature is an intelligent 
Crop Recommendation System powered by Machine Learning.

CROP RECOMMENDATION SYSTEM - DETAILED IMPLEMENTATION
================================================================================

1. MACHINE LEARNING MODEL
--------------------------------------------------------------------------------
   ALGORITHM: Random Forest Classifier
   
   WHY RANDOM FOREST?
   ------------------
   ✓ Handles non-linear relationships well
   ✓ Resistant to overfitting
   ✓ Works well with multiple features
   ✓ Provides feature importance
   ✓ No need for feature scaling
   ✓ Handles missing values better than other algorithms
   ✓ Ensemble method (combines multiple decision trees)

   MODEL PARAMETERS:
   -----------------
   - n_estimators: 100 (number of decision trees)
   - random_state: 42 (for reproducibility)

   INPUT FEATURES (7 parameters):
   -------------------------------
   1. N (Nitrogen) - Soil nitrogen content (kg/ha)
   2. P (Phosphorus) - Soil phosphorus content (kg/ha)
   3. K (Potassium) - Soil potassium content (kg/ha)
   4. Temperature - Average temperature (°C)
   5. Humidity - Relative humidity (%)
   6. pH - Soil pH level (0-14 scale)
   7. Rainfall - Annual rainfall (mm)

   OUTPUT:
   -------
   - Recommended crop name (e.g., "Rice", "Wheat", "Cotton")

   DATASET:
   --------
   - File: Crop_recommendation.csv
   - Contains historical data of soil parameters and suitable crops
   - Used for training the model

2. MODEL TRAINING PROCESS
--------------------------------------------------------------------------------
   STEP 1: Data Loading
   ---------------------
   - Load CSV file using pandas
   - Dataset contains soil parameters and crop labels

   STEP 2: Feature Selection
   --------------------------
   - X (Features): N, P, K, temperature, humidity, ph, rainfall
   - y (Target): crop label

   STEP 3: Data Splitting
   -----------------------
   - Training set: 80% of data
   - Testing set: 20% of data
   - Random state: 42 (ensures same split every time)

   STEP 4: Model Training
   -----------------------
   - Algorithm: RandomForestClassifier
   - Fits the model on training data
   - Creates 100 decision trees

   STEP 5: Model Evaluation
   -------------------------
   - Predicts on test set
   - Calculates accuracy score
   - Typical accuracy: 95%+ (very high)

   STEP 6: Model Serialization
   ----------------------------
   - Save trained model using joblib
   - File: crop_recommendation_model.pkl
   - Can be loaded later for predictions

3. PREDICTION WORKFLOW
--------------------------------------------------------------------------------
   FRONTEND (User Input):
   ----------------------
   1. User fills form with 7 parameters
   2. Form validation ensures all fields are filled
   3. Data sent to backend via POST request to /api/crop/predict

   BACKEND (Processing):
   ---------------------
   1. FastAPI receives request
   2. Pydantic validates input data
   3. Convert input to numpy array: [[N, P, K, temp, humidity, ph, rain]]
   4. Load pre-trained model from .pkl file
   5. Model predicts crop: model.predict(input_array)
   6. Return prediction as JSON response

   FRONTEND (Display Result):
   --------------------------
   1. Receive prediction from backend
   2. Display recommended crop to user
   3. Show success toast notification

4. API ENDPOINTS
--------------------------------------------------------------------------------
   Crop Recommendation Endpoint:
   -----------------------------
   - URL: POST /api/crop/predict
   - Request Body (JSON):
     {
       "N": 90,
       "P": 42,
       "K": 43,
       "temperature": 20.8,
       "humidity": 82,
       "ph": 6.5,
       "rainfall": 202.9
     }
   
   - Response (JSON):
     {
       "recommended_crop": "Rice"
     }

   - Error Handling:
     * 500: Model not loaded
     * 500: Prediction error with details
